# 功能規格

> Version：V3.0

---

# 一、目的

本文件定義 Travel Companion V3 各旅行工具（Travel Tool）的產品功能規格。

每個旅行工具均依附於 Trip（旅程）之下，並遵循統一的設計規格。

本文件描述的是產品功能與規格，不負責記錄開發進度；開發版本與完成狀態請參閱《02_產品開發路線圖》。

---

# 二、旅行工具定義

旅行工具（Travel Tool）係指依附於 Trip（旅程）之下，提供單一產品功能的功能模組。

目前已規劃的旅行工具：

- 旅程資訊
- 記帳本
- 共同清單

未來將依產品需求持續新增新的旅行工具。

---

# 三、Trip 架構

```text
Trip（旅程）
│
├── 旅程資訊
│
├── 記帳本
│
└── 共同清單
```

所有旅行工具皆依附於 Trip。

---

# 四、旅行工具共通規格

所有旅行工具均須定義以下內容：

| 項目 | 說明 |
|------|------|
| 功能說明 | 此旅行工具的主要用途。 |
| 儲存位置 | localStorage、IndexedDB、Supabase 等。 |
| 資料擁有者 | Trip 或個人。 |
| 適用對象 | 哪些使用者可使用此旅行工具。 |
| 權限規則 | Viewer、Editor、Admin 等權限規則。 |
| 是否可編輯 | 是否允許修改內容。 |
| 同步方式 | 不同步、本機同步、雲端同步等。 |
| 生命週期 | 建立、保存與刪除時機。 |

# 五、旅程資訊

## 功能說明

管理旅程本身的基本資訊。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | Supabase |
| 資料擁有者 | Trip |
| 適用對象 | Trip 成員 |
| 權限規則 | 依 Trip 權限管理 |
| 是否可編輯 | 目前僅建立旅程時建立，後續依功能規劃調整 |
| 同步方式 | 雲端同步 |
| 生命週期 | 建立 Trip 時建立，刪除 Trip 時一併刪除 |

## 管理資料

- Trip
- Trip Member
- Itinerary（未來）

## 功能能力

### 基本資料

- 旅程名稱
- 日期
- 地點

### 成員管理

- 共用旅程
- 管理旅程成員

### 行程管理（未來）

- 每日行程
- 行程排序
- 行程時間

# 六、記帳本

## 功能說明

管理旅程內所有收支資料及附件。

## 旅行工具規格

| 項目 | 內容 |
|------|------|
| 儲存位置 | IndexedDB／Supabase（依權限） |
| 資料擁有者 | Trip（一般 User 使用本機資料） |
| 適用對象 | Admin、Editor、User |
| 權限規則 | 依 Trip 權限管理 |
| 是否可編輯 | 可（依權限） |
| 同步方式 | Offline First，依權限同步 |
| 生命週期 | 建立 Trip 後即可使用，刪除 Trip 時依規則移除 |

## 管理資料

- Expense
- Attachment

## 功能能力

### 帳務管理

- 新增帳目
- 修改帳目
- 刪除帳目
- 修改記帳日期
- 日期排序
- 日期分頁
- 多幣別
- 付款人

### 附件管理

- 新增附件
- 修改附件
- 刪除附件
- 開啟附件
- 自動更新附件連結

### 同步管理

- Offline First
- Pending Queue
- Pending Badge
- 手動同步
- 自動同步
- 帳目同步
- 附件同步

### 匯出

- Excel 匯出
- 匯出記帳日期
- 匯出附件網址
