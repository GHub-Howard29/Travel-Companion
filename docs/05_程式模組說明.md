# 程式模組說明

目前系統已完成 V2.0 重構。

原本集中於 App.tsx 的功能，已逐步拆分為獨立模組，方便後續維護及功能擴充。

---

# 模組架構

App.tsx
│
├── Components（畫面）
├── Hooks（React Hook）
├── Actions（帳本操作）
├── Storage（本機資料）
├── Sync（同步）
├── Attachment（附件）
├── Services（服務）
├── Utils（工具）
└── Types（型別）

---

# App.tsx

主要負責：

- 畫面控制
- React State
- 呼叫各模組
- 管理 UI 流程

不直接處理資料儲存或同步。

---

# Actions

負責帳本操作。

主要內容：

- 新增帳目
- 修改帳目
- 刪除帳目
- 更新 Pending
- 呼叫 Storage
- 呼叫 Sync

每個 Action 只負責一項工作。

---

# Storage

負責本機資料管理。

主要模組：

### expenseStorage.ts

功能：

- 帳目資料保存
- 帳目讀取
- 更新帳目
- 刪除帳目

---

### attachmentStorage.ts

功能：

- 附件保存
- 附件讀取
- 附件刪除
- 附件更新

---

### attachmentDb.ts

功能：

- IndexedDB 操作
- Blob 儲存
- Blob 讀取

---

# Sync

負責雲端同步。

主要模組：

### syncManager.ts

功能：

- 上傳資料
- 下載資料
- Pending Queue
- 同步流程控制

---

# Attachment

負責附件管理。

主要模組：

### attachmentService.ts

功能：

- 上傳附件
- 下載附件
- 更新附件
- 刪除附件
- Storage 同步

附件與帳目採分開同步。

---

# Services

負責各項商業邏輯。

目前包含：

### tripService.ts

功能：

- 旅程管理
- 共用帳本
- 個人帳本

未來：

- Reference
- Checklist

亦將獨立成 Service。

---

# Utils

提供共用工具函式。

例如：

- 日期格式
- 金額格式
- 附件處理
- 共用 Helper

避免重複程式碼。

---

# Types

集中管理 TypeScript 型別。

例如：

- ExpenseItem
- Attachment
- Trip
- User
- PendingItem

所有模組共用同一份型別。

---

# 模組設計原則

1. 一個模組只負責一項工作。
2. App.tsx 不直接處理資料存取。
3. Storage 不直接操作 UI。
4. Sync 不直接操作畫面。
5. Utils 不包含商業邏輯。
6. Type 統一集中管理。
