# 權限設定資料表

本文件是 Travel Companion 目前權限設定的集中對照表。程式實作以
`src/permissions/permission.ts`、`src/permissions/roleMapper.ts` 為準；本文件供產品確認、測試與後續調整時查閱。

最後更新：2026-07-24  
狀態：目前程式設定；V3.3.1 相關卡片排序與每日行程排序尚待 Product Owner 回歸驗證，尚未發布。

---

## 一、角色判定

| 角色 | 判定方式 | 範圍 |
|---|---|---|
| `guest` | 未登入 | 所有旅程皆為訪客視角 |
| `user` | 已登入，但不是目前旅程的編輯者 | 僅一般使用權限 |
| `trip_editor` | 已登入，且 `admin_users` 內有目前 Trip 的 `trip_editor` 指派 | 僅被指派的旅程 |
| `super_admin` | 已登入，且 `admin_users` 內有 `super_admin` 身分 | 所有旅程 |

`trip_editor` 切換至未被指派的旅程時，會以 `user` 權限使用該旅程。

| 功能 | guest | user | trip_editor（目前旅程） | super_admin |

| 領隊／導遊聯絡資訊或自駕租車資訊 | 可瀏覽 | 可瀏覽 | 可管理、同步雲端 | 可管理、同步雲端 |
| 其他資訊 | 可瀏覽符合資料角色設定的卡片 | 可瀏覽符合資料角色設定的卡片 | 可管理、同步雲端 | 可管理、同步雲端 |

## 三、私人確認清單的資料隔離

私人確認清單以「登入使用者 + Trip」為獨立範圍：

- `guest` 沒有入口，也不能使用。
- 已登入的 `user` 可建立與管理自己的本機私人清單，但不會同步雲端。
- `trip_editor` 與 `super_admin` 可同步自己的私人清單到雲端。
- 雲端以 `owner_user_id = auth.uid()` 限制讀寫；每位使用者只能讀寫自己的私人清單。
- `super_admin` 的旅程管理權不包含檢視、編輯或同步其他使用者的私人清單。
- 私人清單勾選後，畫面會自動將已勾選項目置底；這不會影響其他使用者的清單。

## 四、資料與同步範圍

| 資料類型 | 雲端資料範圍 | 本機／個人隔離規則 |
|---|---|---|
| 每日行程、領隊／自駕資訊、其他資訊 | 由目前 Trip 的編輯者／管理員維護 | guest / user 僅讀取可見資料 |
| 共同檢查清單 | 由目前 Trip 的編輯者／管理員維護共用雲端版本 | user 使用帳號＋Trip 分隔的本機副本；不影響共用雲端版本 |
| 共用帳本 | 目前 Trip 的共用帳本 | user 使用自己的個人本機帳本 |
| 外幣換算 | 目前 Trip 的編輯者／管理員共用雲端歷史 | guest / user 僅用本機換匯資料 |
| 私人確認清單 | 使用者自己的私有雲端資料 | 依 `userEmail + tripId` 分開保存；不同使用者互不影響 |

## 五、維護規則

- 調整角色、管理按鈕、同步範圍或 Supabase RLS（資料列權限）時，必須同步更新本文件。
- 產品規格以本表為溝通基準；實際執行仍以 `permission.ts` 與 Supabase RLS 為準。
- 若表格與程式不一致，先修正不一致處，再進行發布。
