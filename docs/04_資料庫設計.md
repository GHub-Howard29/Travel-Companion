> Version：V3.0
>

# 一、目的

本文件說明 Travel Companion V3 的資料儲存架構。

主要描述：

- 資料儲存位置
- 資料同步方式
- 資料表設計
- 資料關聯

本文件不討論產品功能，也不討論程式實作。

---

# 二、資料儲存架構

Travel Companion 採用 Offline First（離線優先）架構。

資料依用途儲存在不同位置。

```text
Travel Companion
        │
        ▼
     Storage Layer
        │
        ├── localStorage
        ├── IndexedDB
        └── Supabase
```

所有資料皆透過 Storage Layer 存取。

Travel Tool 不直接操作資料來源。

---

# 三、本機資料

目前使用：

## localStorage

用途：

- 使用者設定
- UI 設定
- 快取資訊

特色：

- 容量小
- 存取快速
- 不同步

---

## IndexedDB

用途：

- 本機帳本
- 附件
- 離線資料
- 暫存同步資料

特色：

- 容量大
- 支援附件
- 支援離線
- 可與雲端同步

---

# 四、雲端資料

目前使用：

Supabase

用途：

- 旅程
- 成員
- 雲端帳本
- 共同清單
- 私人清單
- 旅行資訊
- 附件

特色：

- 多人共同使用
- 自動同步
- 權限管理

---

# 五、資料同步

資料同步依功能決定。

目前規劃如下：

| 功能 | 本機 | 雲端 |
|------|------|------|
| 本機帳本 | ✓ | ✗ |
| 雲端帳本 | ✓ | ✓ |
| 共同清單 | ✗ | ✓ |
| 私人清單 | ✗ | ✓ |
| 旅行資訊 | ✗ | ✓ |

同步流程由 Foundation 統一管理。

Travel Tool 不直接處理同步。

---

# 六、主要資料表

目前規劃如下：

| 資料表 | 用途 |
|------|------|
| trips | 旅程資料 |
| trip_members | 旅程成員 |
| expenses | 雲端帳本 |
| expense_attachments | 帳本附件 |
| checklists | 清單 |
| checklist_items | 清單項目 |
| references | 旅行資訊 |
| reference_files | 旅行資訊附件 |
| admin_users | 系統管理權限 |

資料表名稱可依實際開發調整。

---

# 七、資料關聯

```text
Trip
│
├── Members
│
├── Expenses
│   └── Attachments
│
├── Checklists
│   └── Checklist Items
│
└── References
    └── Files
```

所有 Travel Tool 的資料皆依附於 Trip。

旅程刪除時，相關資料應一併清除。

---

# 八、未來規劃

未來新增功能時：

例如：

- 預約紀錄
- 行李清單
- 旅行日誌
- 保險資訊

若需要雲端儲存，

則新增對應資料表。

並遵循既有資料架構。

避免修改既有資料表設計。

---

# 九、設計原則

資料庫設計遵循以下原則：

- Trip（旅程）為資料核心。
- Storage Layer 統一管理資料存取。
- Travel Tool 不直接操作資料來源。
- 優先擴充，不修改既有架構。
- 保持低耦合、高可維護性。
