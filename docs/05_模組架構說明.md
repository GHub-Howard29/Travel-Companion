# 模組架構說明

> Version：V3.0
>
> 最後更新：2026-07

---

# 一、目的

本文件說明 Travel Companion V3 各模組的職責。

每個模組皆遵循單一職責原則（Single Responsibility Principle）。

---

# 二、整體架構

```text
Travel Companion
│
├── Foundation
│
├── Trip
│
└── Travel Tools
    ├── Expense
    ├── Checklist
    ├── Reference
    └── ...
```

---

# 三、Foundation

## 主要職責

提供所有 Travel Tool 共用能力。

包含：

- Current Trip
- Current User
- Workspace
- Permission
- Role
- Sync State
- Storage

Foundation 不負責任何商業邏輯。

---

# 四、Trip

## 主要職責

Trip 為產品核心。

負責管理：

- 旅行基本資訊
- 成員
- 權限
- Workspace
- 雲端同步資訊

Trip 不直接管理 Travel Tool。

---

# 五、Expense Module

## 功能

旅行記帳。

包含：

- 新增帳目
- 編輯帳目
- 刪除帳目
- 附件管理
- 匯出 Excel

依賴：

- Foundation

不得依賴其他 Tool。

---

# 六、Checklist Module

## 功能

旅行清單管理。

包含：

- Public Checklist
- Private Checklist

依賴：

- Foundation

不得依賴其他 Tool。

---

# 七、Reference Module

## 功能

旅行資訊整理。

例如：

- 地址
- 網站
- 注意事項
- 備忘錄

依賴：

- Foundation

不得依賴其他 Tool。

---

# 八、未來模組

預計新增：

- Reservation
- Packing
- Travel Journal
- Insurance
- AI Assistant

新增模組時：

不得修改既有模組。

---

# 九、模組設計原則

每個模組：

- 自己管理 UI
- 自己管理 State
- 自己管理 Business Logic

共同能力：

由 Foundation 提供。

---

# 十、資料流

```text
UI
│
▼
Travel Tool
│
▼
Foundation
│
▼
Storage
│
▼
Supabase
```

所有資料流皆遵循此架構。

---

# 十一、長期目標

每新增一個 Travel Tool，

都應能獨立：

- 開發
- 測試
- 維護
- 擴充

避免影響其他模組。
